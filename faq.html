FAQ

Q: Which hypervisor does DVM support?

DVM is Hypervisor Agnostic. Currently, DVM supports KVM and Xen, with VMware ESX in the roadmap.

Q: How is DVM different from CoreOS?

CoreOS is a minimalist Linux distro built for running app container. Though CoreOS is designed primarily for bare metal server, it is commonly used as Guest OS in the cloud, due to the lack of isolation. DVM, on the other hand, eliminates the need of Guest OS, by isolating apps in separated kernels.

Q: Does DVM use container to run the app image?

No, DVM doesn't use container (Docker daemon, LXC, Cgroup, Namespace) technology, except the mnt namespace if Pod is used. DVM is 100% virtualization-based solution, which benefits from the hardware-enforced security in hypervisor.

Q: What kernel version is available in DVM?

Currently, the default HyperKernel in DVM is based on 4.0.1. You can also configure DVM to use any compatible Linux kernel, though they might be slow to boot and require more RAM.
